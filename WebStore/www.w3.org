            <span class="expand_section">Call for Review: HTML5 Proposed Recommendation Published</span>
         </h3>
         <p class="date">

  </p>
      </div>
      <div class="description expand_description"><p>The <a href="http://www.w3.org/html/wg/">HTML Working Group</a> has published a Proposed Recommendation of <a href="http://www.w3.org/TR/2014/PR-html5-20140916/">HTML5</a>. This specification defines the 5th major revision of the core language of the World Wide Web: the Hypertext Markup Language (HTML). In this version, new features are introduced to help Web application authors, new elements are introduced based on research into prevailing authoring practices, and special attention has been given to defining clear conformance criteria for user agents in an effort to improve interoperability. Comments are welcome through 14 October. Learn more about the <a href="http://www.w3.org/MarkUp/Activity">HTML Activity</a>.</p>

            <span class="expand_section">Requirements for Latin Text Layout and Pagination Draft Published</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2014-09-30T15:45:12Z"> 30 September 2014 </span> | <a title="Archive: Requirements for Latin Text Layout and Pagination Draft Published" href="http://www.w3.org/blog/news/archives/4106">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>The <a href="http://www.w3.org/dpub/IG/">Digital Publishing Interest Group</a> has published a Working Draft of <a href="http://www.w3.org/TR/2014/WD-dpub-latinreq-20140930/">Requirements for Latin Text Layout and Pagination</a>. This document describes requirements for pagination and layout of books in latin languages, based on the tradition of print book design and composition. It is hoped that these principles can inform the pagination of digital content as well, and serve as a reference for the CSS Working Group and other interested parties. This work was inspired by the Requirements for Japanese Text Layout. Learn more about the <a href="http://www.w3.org/dpub/">Digital Publishing Activity</a>.</p>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">Seeking Wide Review: TTML Text and Image Profiles for Internet Media Subtitles and Captions 1.0</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2014-09-30T15:36:23Z"> 30 September 2014 </span> | <a title="Archive: Seeking Wide Review: TTML Text and Image Profiles for Internet Media Subtitles and Captions 1.0" href="http://www.w3.org/blog/news/archives/4104">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>The <a href="http://www.w3.org/AudioVideo/TT/">Timed Text Working Group</a> has published a Working Draft of <a href="http://www.w3.org/TR/2014/WD-ttml-imsc1-20140930/">TTML Text and Image Profiles for Internet Media Subtitles and Captions 1.0</a>. This document specifies two profiles of Timed Text Markup Language 1: a text-only profile and an image-only profile. These profiles are intended to be used across subtitle and caption delivery applications worldwide, thereby simplifying interoperability, consistent rendering and conversion to other subtitling and captioning formats. The text profile is a superset of TTML Simple Delivery Profile for Closed Captions (US). The review period ends on <strong>27 October 2014</strong>. Learn more about the <a href="http://www.w3.org/2008/WebVideo/">Video in the Web Activity</a>.</p>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">Document Object Model (DOM) Level 3 Events Specification Draft Published</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2014-09-25T18:27:21Z"> 25 September 2014 </span> | <a title="Archive: Document Object Model (DOM) Level 3 Events Specification Draft Published" href="http://www.w3.org/blog/news/archives/4098">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>The <a href="http://www.w3.org/2008/webapps/">Web Applications Working Group</a> has published a Working Draft of <a href="http://www.w3.org/TR/2014/WD-DOM-L
evel-3-Events-20140925/">Document Object Model (DOM) Level 3 Events Specification</a>. DOM Events is designed with two main goals. The first goal is the design of an event system which allows registration of event listeners and describes event flow through a tree structure. Additionally, the specification will provide standard modules of events for user interface control and document mutation notifications, including defined contextual information for each of these event modules. The second goal of DOM Events is to provide a common subset of the current event systems used in existing browsers. This is intended to foster interoperability of existing scripts and content. Learn more about the <a href="http://www.w3.org/2006/rwc/">Rich Web Client Activity</a>.</p>
</div>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">Last Call: CSS Flexible Box Layout Module Level 1</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2014-09-25T18:27:21Z"> 25 September 2014 </span> | <a title="Archive: Last Call: CSS Flexible Box Layout Module Level 1" href="http://www.w3.org/blog/news/archives/4100">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>The <a href="http://www.w3.org/Style/CSS/members">Cascading Style Sheets (CSS) Work
ing Group</a> has published a Last Call Working Draft of <a href="http://www.w3.org/TR/2014/WD-css-flexbox-1-20140925/">CSS Flexible Box Layout Module Level 1</a>. The specification describes a CSS box model optimized for user interface design. In the flex layout model, the children of a flex container can be laid out in any direction, and can &#x201C;flex&#x201D; their sizes, either growing to fill unused space or shrinking to avoid overflowing the parent. Both horizontal and vertical alignment of the children can be easily manipulated. Nesting of these boxes (horizontal inside vertical, or vertical inside horizontal) can be used to build layouts in two dimensions. Comments are welcome through 25 October. Learn more about the <a href="http://www.w3.org/Style/">Style Activity</a>.</p>

            <span class="expand_section">User Agent Accessibility Guidelines (UAAG) 2.0 Working Drafts Updated</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2014-09-25T18:27:20Z"> 25 September 2014 </span> | <a title="Archive: User Agent Accessibility Guidelines (UAAG) 2.0 Working Drafts Updated" href="http://www.w3.org/blog/news/archives/4102">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>The User Agent Accessibility Guidelines Working Group (<a href="http://www.w3.org/WAI/UA/">UAWG</a>) today published updated Working Drafts of <a href="//www.w3.org/TR/2014/WD-UAAG20-20140925/">User Agent Accessibility Guidelines (UAAG) 2.0</a> and <a href="//www.w3.org/TR/2014/WD-UAAG20-Reference-20140925/">UAAG 2.0 Reference: Explanations, Examples, and Resources for User Agent Accessibility Guidelines 2.0</a>. UAAG defines how browsers, media players, and other &#x201C;user agents&#x201D; should support accessibility for people with disabilities and work with assistive technologies. It is introduced in the <a href="http://www.w3.org/WAI/intro/uaag">UAAG Overview</a>. Read about the <a href="http://www.w3.org/WAI/">Web Accessibility Initiative (WAI)</a>.</p>

         <h3 class="h4 tPadding0 bPadding0 summary">
            <span class="expand_section">Upcoming Workshop: Privacy and User&#x2013;Centric Controls</span>
         </h3>
         <p class="date">
            <span class="dtstart published" title="2014-09-22T19:28:23Z"> 22 September 2014 </span> | <a title="Archive: Upcoming Workshop: Privacy and User&#x2013;Centric Controls" href="http://www.w3.org/blog/news/archives/4089">Archive</a>
         </p>
      </div>
      <div class="description expand_description"><p>Today the W3C announced a <a href="https://www.w3.org/2014/privacyws/">Workshop on Privacy and User&#x2013;Centric Controls</a>, on 20&#x2013;21 November 2014, Berlin, Germany to investigate strategies toward better privacy protection on the Web, especially for mobile devices.</p>
<p>The Workshop focus will be on users: user experience, user behavior and how we can offer controls that provide the necessary transparency of privacy-affecting interactions. We will also discuss how developers can meet users&#x2019; privacy needs on the Web, including what APIs are necessary for user privacy.</p>
<p>Sample Workshop topics include (but are not limited to):</p>
<ul class="show_items">
<li>Improving the UI for stateful services, overview of states</li>
<li>Defaults for expiration of stateful situations</li>
<li>How to convey state information to the User</li>
<li>How to deal with logging and how to provide interfaces for logged data</li>
<li>Requirements for private browsing on mobile</li>
<li>A privacy ontology for mobile apps and their use of personal data</li>
<li>The value of privacy in paradigms for mobile UI</li>
<li>Selective release of personal information to apps</li>
<li>Controlling the geo-location interfaces, including UI challenges</li>
<li>Enforcing data expiry</li>
</ul>
<p>W3C membership is not required to participate. There is no cost to participate and the event is open to all. All participants are required to submit a position paper or statement of interest by 10 October 2014. Learn more <a href="http://www.w3.org/2014/privacyws/participation.html">how to participate</a>.</p>
